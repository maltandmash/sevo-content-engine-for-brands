The user wants their content status.

1. Read `content-status.md` from the brand folder (a `brand/` folder in the working folder, or the company brand folder the engine has been pointed at). If you cannot find a brand folder, the brand is not set up yet, so tell them to run `/voice-pack` first.
2. Report in short:
   - What is published, newest first
   - What is queued, in priority order, with the next item called out
   - Any topic clusters in progress and gaps in them
   - Anything stuck waiting on a reviewer or on approved-facts
3. End with a one-line "What I'd do next", usually the single highest-value next piece to write.
